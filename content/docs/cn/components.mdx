# API Reference: System Core

Welcome to the official developer documentation for the **Core Engine**. This guide provides the necessary endpoints and data schemas required to integrate with our distributed architecture.

## Getting Started

To begin using the API, you must first obtain an `API_KEY` from your dashboard. All requests must be sent over HTTPS.

<Warning>
  **Important:** The current version (2.4.0) deprecates the legacy `/v1/auth` endpoint. Please migrate to `/v2/identity` immediately.
</Warning>

## Authentication

We use Bearer Token authentication. Ensure your headers look like the following:

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json
```

### Response Schema

The system returns a standard JSON object for all successful requests:


| Property | Type | Description |
| :--- | :--- | :--- |
| `status` | string | Either `success` or `error`. |
| `timestamp` | integer | Unix timestamp of the request. |
| `data` | object | The payload of the response. |

## Advanced Usage

### Recursive Data Fetching

When dealing with nested tree structures, use the recursive query parameter. Note the performance impact on deep nests.

```javascript
const fetchData = async (depth = 3) => {
  const response = await fetch(`https://core-engine.io{depth}`);
  const json = await response.json();
  
  if (json.status !== 'success') {
    throw new Error('System Overload');
  }
  
  return json.data;
};
```

## Error Handling

The API uses standard HTTP response codes.

1. **400 Bad Request**: The request was unacceptable, often due to missing a required parameter.
2. **401 Unauthorized**: No valid API key provided.
3. **429 Too Many Requests**: You have hit the rate limit (1,000 req/min).
